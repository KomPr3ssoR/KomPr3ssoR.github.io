Threats

  • The term “security” encompasses host- and network-based security, and physical security
  • Common threats include man-in-middle attacks, DoS, and DDoS attacks
  • Symptoms of such attacks include renamed system files, missing files, and more.

Physical Security

Objective 2.1 Summarize the importance of physical security measures

  • Physical security includes perimeter security, room locks, and individual device security
  • Security guards and mantraps help perimeter security
  • Locks, badges, smart cards, and biometrics enable room-level security
  • Device security devices include server locks, USB locks, and screen filters

Passwords and Authentication

Objective 2.5 Given a scenario, implement security best practices to secure a workstation

  • Modern password security relies on passwords and hashes
  • Methods for cracking passwords and hashes include brute force, dictionary attacks,and rainbow tables
  • Protect with best password practices, such as upper- and lower-case letters, numbers, and non-alphanumeric passwords; long passwords help, too

  • Malware infestations manifest as browser redirection, application crashes, update failures, and more

Anti-Malware

Objective 3.3 Given a scenario, use best practice procedures for malware removal

Objective term

Anti-Malware

Recovery console

Backup/Restore

End-user education

Software firewalls

Secure DNS

CompTIA 7-Step

  1. Identify and research malware symptoms
  2. Quarantine the infected systems
  3. Disable System Restore (in Windows)
  4. Remediate the infected systems (a. Update the anti-malware softwares b. Scan and use removal techniques in safe mode, pre-installation environment)
  5. Schedule scans and run updates
  6. Enable System restore; also and create a restore point (in WIndows)
  7. Educate the end user
  • Prepare for malware attacks with good backups, end-user education, and firewalls
  • Use secure DNS options, such as a trusted third-party site and encryption
  • Memorize the CompTIA A+ 7-step malware-recovery process

Social Engineering

Objective 2.5 Compare and contrast social engineering, threats, and vulnerabilities

Objective term

Phishing

Spear phishing

  • Social engineering enables bad people to use information from people inside to gain access to sensitive information
  • Social engineering attacks include telephone scams, tailgating, shoulder surfing, and dumpster diving
  • Phishing and spear phishing attacks seek personal or financial information with targeted scams

The Internet

The Internet

Objective 4.9 Given a scenario, use remote access technologies

Telnet and SSH

Objective Term:

Telnet - Port 23

Secure Shell (SSH) - port 22

  • The Telnet and the Secure Shell (SSH) protocols provide command line access to remove systems
  • Telnet runs on TCP port 23; SSH runs on TCP port 22
  • Telnet is unencrypted; SSH is encrypted

Remote Desktop Connections

Objective Term:

Remote Desktop Protocol (RDP) - Port 3389

Virtual Network Computing (VNC) - Port 5900+N

  • Windows uses Remote Desktop Protocol (RDP) for remote connectivity
  • Use Virtual Network Computing (VNC) protocol for connecting to Linux and Mac systems
  • Try Tight VNC to use VNC from a Windows system

The World Wide Web

Objective 2.2 Explain logical security concepts

Objective Term:

HTTP - port 80 (insecure)

HTTPS - port 443 (encrypts everything)

Secure Sockets Layer (SSL)

Transport Layer Security (TLS)

  • The World Wide Web uses either the Hypertext Transfer Protocol (HTTP) or HTTP Secure (HTTPS) protocols
  • HTTP uses port 80 and is not secure
  • HTTPS uses port 443 and uses certificates to make a secure connection
  • Make sure to recognize typical certificate errors

Wireless Networking

Objective 2.3 Compare and contrast wireless security protocols and authentication methods

Wireless Encryption

Objective Term:

WEP (Wired Equivalent Privacy)

TKIP (Temporal Key Integrity Protocol)

AES (Advanced Encryption Standard)

WPA (Wi-Fi Protected Access)

WPS (Wi-Fi Protected Setup)

  • The obsolete WEP encryption is easily cracked today and is never used
  • WPA introduced TKIP-personal shared key and RADIUS authentication
  • WPA2 added AES encryption
  • WPS automates encryption, but is easily crackable

Enterprise Wireless

Object Term:

Power over Ethernet (PoE)

PoE (1st generation) and PoE+

PoE injector

AAA - authorization, authentication, accounting

RADIUS or TACACS+

Extended SSID (ESSID)

  • Enterprise WAPs often use Power over Ethernet (PoE)
  • Use powerful wireless analysis tools to determine WAP location
  • Enterprise networks often use RADIUS or TACACS+ protocols for authentication
  • Two or more WAPs sharing the same SSID are known as ESSIDs

Port Forwarding

Posted on 2020-06-24 In Notes

Port Forwarding

  • A port forward is when a NAT router opens incoming traffic on a certain port access to a single system on the internal network
  • You configure port forwarding on the NAT router
  • We often use nonstandard port numbers for security reasons
  • We use Dynamic DNS to give DNS names to port forwarded devices

Host Firewall

  • Host firewall is an individual firewall. Windows Defender Firewall is a host-based firewall that comes with Windows
  • Host firewalls protect systems using host features such as file names or process IDs
  • Windows defender firewall has three settings:
    • Domain network
    • Private network
    • Guest or public network

Network discovery & exception

  • You can create exceptions manually if needed
